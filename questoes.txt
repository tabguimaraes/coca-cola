O que é um repositório no Git e qual a sua importância?
No Git, ele é um repositório local para armazenar o projeto e o histórico de alterações (versionamento). O GitHub é uma rede social e espaço colaborativo que usa o Git como plataforma.


Qual a função do Git Bash?
É um terminal próprio para executar comandos do git.


O que faz o comando git add e qual a diferença entre git add <arquivo> e git add .?
Git add adiciona os arquivos alterados para serem "comitados". git add <arquivo> adiciona apenas o arquivo enquanto o git add . adiciona todos os arquivos que estão aguardando na árvore de alterações.


Por que utilizamos o ponto (.) no comando git add .?
o git add . adiciona de uma úbica vez todos os arquivos que estão aguardando na árvore de alterações.

Explique o comando git commit e por que é importante criar uma mensagem descritiva.
Ele "carimba" uma tag com a descrição das alterações que foram executadas naquele(s) arquivo(s) naquela data. Por isso, é preciso ser coerente e suscinto na mensagem para que ela seja clara e objetiva sobre qual a mudança executada.


Qual a função do comando git push?
Envia as alterações após elas serem "comitadas" e a partir deste ponto, as alterações são salvas na linha do tempo do projeto.


Qual a importância de criar commits frequentes durante o desenvolvimento de um projeto?
Facilita a identificação de alguma alteração que possa ter causado um bug, por exemplo. Assim, evita-se que seja necessário retornar para um momento muito distante do atual, causando retrabalho. 


Explique a função das tags HTML e como elas estruturam uma página.
As tags servem para defirnir a estrutura da página que está sendo criada, independente do conteúdo, garantindo padronização, desde que sigam boas práticas de desenvolvimento, como o uso de html semãntico.


O que são atributos em uma tag HTML e qual a sua função?
Servem para definir informações as tags, como o localização de uma imagem dentro da pasta do projeto (src="./imagem.jpg")


O que é o Emmet e como ele facilita o desenvolvimento de páginas HTML?
O Emmet possui blocos de código que costumam ser muito utilizados, agilizando o desenvolvimento e evitando erros ao livrar o desenvolvedor de repetição de códigos o tempo todo.


O que é o box-sizing no CSS e por que usamos ele no reset inicial?
Ele calcula a altura e largura de um elemento com mais precisão quando inserido no reset inicial, pois é possível evitar que o padding e margin aumentem as medidas dos eelementos

Por que é importante utilizar tags semânticas no HTML?
Os leitores de tela identificam as tags semanticas e podem se orientar na leitura do conteudo adequado para o usuario com necessidade de leitura assistida.


O que são media queries no CSS e qual a sua função?
São pontos de quebra de página, baseadas em critérios como altura e largura da tela, tipo dispositivo, etc. Isso garante que a página seja legível em qualquer tipo de dispositvo compatível.


Qual a função do GitHub Pages?
Fazer o deploy da página que foi criada, mantendo ela "no ar" e visível por qq usuário da internet.


O que é o Flexbox no CSS e para que ele é utilizado?
É um recurso para criação de designs responsiveis devido ao fato de ele criar elementos "flexíveis"


Qual a diferença entre justify-content e align-items em um container Flexbox?
Nunca entendi muito bem o uso destes dois itens, mas acredito que o justify alinhe horizontalmente e o o align na vertical


Como o flex-grow funciona e qual o seu propósito no layout Flexbox?
Na Mozilla diz que: "Define quanto um item pode crescer em relação aos outros dentro do container flexível.", mas na prática não faço idéia de como ou quando utilizar ele.

Explique a diferença entre flex-direction: column; e flex-direction: row;.
flex-direction: column >> cria colunas e os objetos ficam empilhados, uns sobre os outros.
flex-direction: row >> cria linhas e os objetos ficam alinhados lado a lado, até o final do espaço delimitado.
